## Reflection

### 1. What went well during this unit?

What went well during this unit was that our group was able to move from mostly planning into actual development. We set up our project board, created issues, and organized tasks for the next few weeks so the work is clearer and easier to track now. We also made progress on both the frontend and backend, including getting the local app running and starting work on features like product pages, user authentication, category filtering, and product search.

### 2. What were some challenges your group faced in this unit?

One challenge was making sure we did not overlap too much while multiple people were working on different parts of the project at the same time. Another challenge was figuring out what was already implemented, what was still using mock data, and what still needed to be connected to the backend. We also ran into some setup and debugging issues while getting the database connection and local app running correctly.

### Did you finish all of your tasks in your sprint plan for this week? If you did not finish all of the planned tasks, how would you prioritize the remaining tasks on your list?

Yes, we completed the tasks we planned for this unit. We created the project board, added issues, organized tasks for the next few weeks, and started development on the app. Since the board includes work planned for upcoming units as well, not everything on the board was meant to be finished this week. Our focus this unit was on getting organized and making sure development was moving in the right direction.

### Which features and user stories would you consider “at risk”? How will you change your plan if those items remain “at risk”?

The features that seem most at risk are the more complex ones, especially full cart functionality, checkout, and some of the user account and store manager features once everything needs to be fully connected between the frontend, backend, and database. These are not necessarily behind right now, but they may take more time than simpler features. If any of them start to fall behind, we will focus on finishing the required baseline features first and simplify any non-essential parts so we can make sure the core app flow is fully working.

### 5. What additional support will you need in upcoming units as you continue to work on your final project?

Going forward, we will mostly need support with connecting all parts of the app together and making sure everything works smoothly from frontend to backend to database. We will also need help with debugging, deployment, and checking that all required features are fully working by the end of the project.
